Types of Cyber Attacks

Understand the most common cyber attacks, how they work, their real-world impact, and how to protect against them through practical examples.

Course 2 of 10 📋 Difficulty: Beginner ⏱️ Reading Time: 8–10 Minutes 🏷️ Category: Cyber Security Fundamentals

⚠️ Section 1: Introduction to Threat Vectors

This module analyzes the mechanics behind malicious attack surfaces and unexpected threat vectors targeted by modern adversaries.

Definition of Threat Vectors: A threat vector is a path, route, or exploit channel by which a cybercriminal or malicious actor gains unauthorized access to a network, server, endpoint, or system to deliver a malicious payload or harvest confidential data assets.

1. Zero-Day Attack
  • GTU Exam Definition: An attack that exploits a software flaw before the vendor or developers become aware of it and have a chance to deploy a patch.
  • How it Works: Threat actors discover unpatched bugs in applications, craft precise exploit codes, and hit corporate networks silently before defensive definitions are updated.
  • Real-Life Example: The Stuxnet worm utilized multiple zero-day exploits simultaneously to compromise isolated industrial computers.
  • Warning Signs: Unexplained application crashes, unauthorized background process loops, or unusual outward network synchronization.
  • Prevention & Best Practices: Employ advanced behavior-based heuristics scanning, configure runtime application self-protection, and run zero-trust access controls.
2. Supply Chain Attack
  • GTU Exam Definition: A cyber security compromise targeting the weakest links or upstream software/hardware vendors within a trusted multi-party network ecosystem.
  • How it Works: Instead of hacking a heavily defended corporation directly, adversaries compromise a third-party application developer or tool used by that target.
  • Real-Life Example: The SolarWinds breach inserted backdoor code into legitimate software updates, distributing malware to thousands of trusted systems downstream.
  • Warning Signs: Unexpected behavioral shifts in verified vendor updates or anomalous code signing modifications.
  • Prevention & Best Practices: Strictly audit vendor security credentials, implement rigorous integrity monitoring, and run all third-party code with limited privileges.
3. Insider Threat
  • GTU Exam Definition: A security risk originating from within an organization, usually caused by current or former employees, contractors, or trusted entities who abuse their authorized network access.
  • How it Works: A disgruntled internal actor knowingly exfiltrates sensitive database dumps, or a careless user accidentally misconfigures a production cloud bucket to be publicly accessible.
  • Real-Life Example: A data engineer copying customer databases onto personal external flash drives before resigning.
  • Warning Signs: Accessing sensitive corporate repositories outside normal operating hours or massive volume data downloads.
  • Prevention & Best Practices: Enforce the Principle of Least Privilege (PoLP), execute detailed user behavior analysis, and record comprehensive audit logs.

🎯 Section 2: Malware Attacks (Viruses, Trojans & Ransomware)

An overview of unauthorized system code deployments that disrupt memory environments and user access privileges.

Core Content Formulas and Technical Definitions

Malware is a broad term for malicious software designed to compromise or damage systems. Below is the operational taxonomy of modern malicious code vectors:

1. Virus vs. Worm
  • Definitions & GTU Perspective: A Virus is parasitic code that requires a human to run a host file to execute. A Worm is a self-replicating standalone entity that automatically crawls networks via protocol vulnerabilities.
  • Real-Life Example: The ILOVEYOU script required user execution (Virus), whereas the WannaCry tool automatically swept through exposed SMB network ports (Worm).
  • Mitigation: Isolate macro execution parameters inside office suites and run active host firewall rules.
2. Trojan Horse & Ransomware
  • Definitions & Mechanisms: A Trojan Horse hides its payload inside an innocent file or utility program. Ransomware uses strong cryptographic algorithms (such as AES+RSA) to lock down file trees, demanding financial compensation for decryption keys.
  • Real-Life Example: Downloading a fake driver package that covertly deploys file-locking modules across all shared active drives.
  • Mitigation: Isolate backup volumes from the primary local system mapping and avoid running software binaries from unverified channels.
3. Spyware, Adware & Keyloggers
  • Definitions & Impact: Spyware stealthily tracks browser metrics and configurations. Adware forcefully populates desktops with repetitive popup windows. Keyloggers capture every physical keystroke to steal plain-text passwords.
  • Real-Life Example: A browser extension that silently intercepts credit card values during checkout.
  • Mitigation: Check browser extension permissions regularly and deploy localized anti-spyware engines.
4. Rootkits & Botnets
  • Definitions & Operations: A Rootkit replaces core operating system files to gain deep kernel access while hiding itself from task managers. A Botnet connects thousands of compromised systems into a unified zombie network controlled via a remote server.
  • Real-Life Example: Malicious root files hiding administrative backdoors or zombie clusters launching synchronized credential stuffing attacks.
  • Mitigation: Enforce cryptographically verified Secure Boot sequences and inspect outbound network traffic spikes.
5. Drive-by Download
  • Definition & How it Works: The unintended installation of malicious software on a user's device that occurs without their active interaction, triggered simply by viewing an infected web page.
  • Real-Life Example: Visiting a compromised mainstream blog that uses hidden iframe elements to exploit an old browser script engine flaw.
  • Mitigation: Keep all internet browsers updated and turn off automatic code execution scripts on unknown pages.

🎣 Section 3: Social Engineering & Phishing Mechanics

How malicious operations leverage identity deception to gather credentials and exploit network vectors.

Threat Warning Model & Deception Verification Matrices:

Social Engineering exploits human psychology—using panic, curiosity, or greed—instead of technical bugs. Here are the specific phishing and engineering styles you must protect against:

1. Phishing Taxonomy (Spear Phishing, Whaling)
  • Phishing: Mass email blasts sent out randomly to trick users with generic traps like "Your account is locked."
  • Spear Phishing: Highly customized attacks targeting a specific person, using their actual name, title, and coworker details.
  • Whaling: Spear phishing aimed specifically at high-profile executives like the CEO or CFO to authorize large wire transfers.
  • Real-Life Example: An attacker posing as a supply chain vendor to email a customized, urgent unpaid invoice directly to a billing manager.
2. Alternate Channels (Smishing, Vishing, Quishing)
  • Smishing (SMS Phishing): Urgent text alerts containing links to fake package tracking pages or account recovery portals.
  • Vishing (Voice Phishing): High-pressure phone scams using spoofed caller IDs or AI voice clones to extract banking OTPs.
  • Quishing (QR Code Scams): Swapping real QR codes on public poster frames or restaurant tables with malicious stickers that open phishing sites.
3. Watering Hole Attack
  • GTU Exam Definition: A strategy where an attacker compromises a specific, trusted website frequently visited by a target group or company, rather than launching a direct attack.
  • How it Works: The attacker profiles their target's browsing habits, infects a local site they trust, and waits for them to visit and catch the payload.
  • Real-Life Example: Injecting malicious scripts into a regional defense contractor discussion board to target engineering companies.
  • Mitigation: Implement strict internet gateway filtering and inspect external resource requests carefully.

💥 Section 4: Network Layer Inundation (DoS & DDoS)

Understanding distributed traffic spikes intended to block network services and crash hardware targets.

DoS Attack
Definition: Denial of Service via a single source system stream.
Metrics & Vectors: Employs structured point-to-point traffic protocols like a single high-bandwidth ping-of-death command loop or manual TCP SYN packet flood saturation routines.
Warning Signs: Localized service timeouts, high target memory spikes, and specific single-IP log filling.
Mitigation: Simple source IP blackholing, standard connection rate limiting, and configuring baseline infrastructure firewall blocking metrics.
DDoS Attack
Definition: Distributed Denial of Service using coordinated global botnets.
Metrics & Vectors: Traffic spikes often surge beyond hundreds of gigabits per second, leveraging distributed amplification techniques like DNS reflection or NTP amplification.
Warning Signs: Rapidly skyrocketing network usage, high multi-region request failures, and widespread load balancer drops.
Mitigation: Anycast routing frameworks, cloud-based scrubbing centers, and automated anomaly-driven traffic distribution.

🕵️‍♂️ Section 5: Interception & Technical Exploits (MitM, Injection & Hijacking)

Analyzing unauthorized communication eavesdropping and data manipulation between linked devices.

Mitigation Parameter & Exploit Prevention Models:

Technical network exploits rely on weak input handling, missing encryption, or flawed network trust relationships. Here is how these attacks operate and how to stop them:

1. Man-in-the-Middle (MitM) & Spoofing Frameworks
  • ARP Spoofing: Flooding a local network with fake ARP messages to link an attacker's MAC address with a legitimate gateway's IP address, intercepting local traffic.
  • DNS Spoofing: Poisoning a recursive DNS resolver's cache with fraudulent IP mappings, routing users away from real sites onto fake ones.
  • Session Hijacking: Stealing or intercepting active web session tokens or cookies to impersonate an authenticated user without needing their password.
  • Mitigation: Mandate end-to-end encrypted HTTPS configurations, enforce Static ARP inspection rules, and deploy DNSSEC validation layers.
2. Application Exploits (SQL Injection & Cross-Site Scripting)
  • SQL Injection (SQLi): Inserting malicious SQL statements into web entry forms to bypass login walls, exposing or modifying underlying backend databases.
  • Cross-Site Scripting (XSS): Injecting malicious scripts into trusted websites, which then run inside an unsuspecting visitor's browser to harvest tokens or hijack sessions.
  • Mitigation: Utilize strictly parameterized queries, execute rigid input sanitization, and deploy precise Content Security Policies (CSP).
3. Password Attacks (Brute Force, Dictionary, Credential Stuffing)
  • Brute Force: Systematically testing every possible combination of alphanumeric characters until the correct password is found.
  • Dictionary Attack: Using pre-arranged lists of common words, phrases, and leaked passwords rather than guessing randomly.
  • Credential Stuffing: Automating the injection of leaked username and password pairs from previous third-party breaches across multiple websites, exploiting password reuse.
  • Mitigation: Implement automated account lockout rules, mandate strong Multi-Factor Authentication (MFA), and use cryptographic password hashing salts.

🎯 Deep Dive Insights

✅ Learning Objectives: Identify common exploit classes, understand vector distribution methods, and deploy defensive validation architectures across endpoints.

✅ Key Highlights: Differentiating automated script threats from targeted social exploitation models helps prioritize patch cycles effectively.

✅ Quick Facts: A significant portion of security incidents begin via unauthorized entry points established through unpatched asset applications.

✅ Did You Know?: Ransomware exploits use standard symmetric and asymmetric encryption principles to turn a powerful security tool into an attack weapon.

✅ Best Security Practices: Keep system binaries up to date, isolate unknown file extensions, enforce zero-trust endpoint profiles, and run automated local file system sweeps.

✅ Summary: Recognizing the structural differences between attack types allows groups to balance preventative firewalls with internal containment structures.

✅ Key Takeaways: Attackers always choose the path of least resistance. Hardening your human verification checks closes the most common avenues for network intrusions.
